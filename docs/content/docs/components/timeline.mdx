# Timeline

Timeline component for displaying events, steps, or history with visual status indicators.

```bash
hauk add timeline
```

## Usage

```tsx
import { Timeline } from "./tui/components/timeline";

function App() {
  const items = [
    { id: "1", title: "Order Placed", status: "completed", date: "Dec 20" },
    { id: "2", title: "Processing", status: "completed", date: "Dec 21" },
    { id: "3", title: "Shipped", status: "current", date: "Dec 23" },
    { id: "4", title: "Delivered", status: "upcoming", date: "Dec 25" },
  ];

  return <Timeline items={items} />;
}
```

## Props

| Prop          | Type                           | Default      | Description           |
| ------------- | ------------------------------ | ------------ | --------------------- |
| `items`       | `TimelineItem[]`               | -            | Timeline items        |
| `orientation` | `"vertical"` \| `"horizontal"` | `"vertical"` | Timeline orientation  |
| `showLines`   | `boolean`                      | `true`       | Show connecting lines |

## TimelineItem

```tsx
interface TimelineItem {
  id: string; // Unique identifier
  title: string; // Item title
  description?: string; // Item description
  date?: string; // Timestamp or date label
  status?: "completed" | "current" | "upcoming" | "error";
  icon?: string; // Custom icon
}
```

## Features

- **Status Indicators** - Visual icons for each status
- **Connecting Lines** - Lines connecting timeline items
- **Dual Orientation** - Vertical or horizontal layout
- **Custom Icons** - Override default status icons
- **Descriptions** - Optional descriptions for each item
- **Date Labels** - Show timestamps or dates

## Notes

- Supports completed, current, upcoming, and error statuses
- Vertical orientation is best for detailed timelines
- Horizontal is good for compact step indicators
